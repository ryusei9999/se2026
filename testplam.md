# テスト計画書

| # | テスト項目 | 入力・操作 | 期待する結果 | 実際の結果 | 判定 |
|---|---|---|---|---|---|
| 1 | 予算を設定 | budget 50000 | 今月の予算を 【50,000円】 に設定しました。 と表示されること |  |  |
| 2 | 予算に文字を入力 | budget abc | エラー: 予算には数値を入力してください。 となり処理が止まること |  |  |
| 3 | 支出を追加（カテゴリ指定） | buy 1200 ランチ 食費 | 追加した（本日合計: 1,200円） と表示され、データが保存されること |  |  |
| 4 | 支出を追加（カテゴリ省略） | buy 8500 飲み会 | カテゴリ入力を促す対話モードになり、入力後に本日合計金額が表示されること |  |  |
| 5 | 金額に文字を入力 | buy abc ランチ 食費 | エラー: 金額には数値を入力してください。 となり処理が止まること |  |  |
| 6 | 今月の支出一覧を表示 | list | 今月の年月（当月）のデータのみがテーブル形式で正しく一覧表示されること |  |  |
| 7 | 収支サマリーを表示 | summary | 設定予算、支出合計、予算残高（🟢/🔴）、カテゴリ別内訳（降順）が正しく表示されること |  |  |
| 8 | 過去すべての履歴を表示 | history | 月の制限なく、過去に登録されたすべての支出データが一覧表示されること |  |  |
| 9 | 指定したIDの支出を削除 | rm 1 | ID 1 のデータを削除しました。 と表示され、対象データが削除されること |  |  |
| 10 | 存在しないIDを指定 | rm 999 |  エラー: ID 999 のデータが見つかりません。 となり処理が止まること |  |  |
| 11 | IDに文字を入力 | rm abc | エラー: IDには数値を入力してください。 となり処理が止まること |  |  |
