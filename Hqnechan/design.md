# design.md

## 画面
1. タイトル画面・サーバー選択
2. メインゲーム画面（操作UI・チャット・ステータス表示）
3. インベントリ・メニュー画面

## データ
1. プレイヤーデータ（座標、HP、所持アイテム、スキン情報など）
2. ワールドデータ（ブロック配置、オブジェクトの状態、時間など）
3. ログデータ（チャット履歴、ログイン・ログアウト、システム警告）

## 処理の流れ
操作入力（キー・マウス） → クライアント側での先行予測・確認 → サーバーへの保存・同期 → 画面へのエフェクト・描画表示

## 構成
ゲーム画面（クライアント） → API（パケット送受信） → ゲームサーバ → DB（セーブデータ管理用データベース）
